# 用户指南与使用后说明

## 推荐使用流程

1. 打开应用。
2. 进入“项目设置”，用自然语言说明项目背景。
3. 上传少量项目材料，例如 CSV、Excel、Markdown、SQL 或 JSON。
4. 进入“测试问题集”。
5. 选择“使用 LLM 生成”，填写 API Base、API Key、模型名，然后生成测试题。
6. 如果已经有人工测试集，选择“上传已有测试问题集”，跳过 LLM 生成。
7. 在审核表中编辑、去重、删除低质量问题。
8. 将可用问题标记为“已通过”。
9. 进入“实验运行”，选择外部 API 模式或历史结果导入模式。
10. 如果选择外部 API 模式，系统会自动使用已通过测试题调用 RAG API，不需要再次上传测试问题集。
11. 如果选择历史结果导入，且文件已经包含 RAG 回答，可以勾选“导入后立即评估”。
12. 进入“评估与失败分析”，查看评分并修正失败标签。
13. 进入“实验对比”，比较不同运行。
14. 进入“导出中心”，导出 Excel 或 PDF。

## 快速演示流程

1. 启动：

```powershell
.\run_app.ps1
```

2. 使用默认示例项目“电商订单问答评测示例”。
3. 在“测试问题集”中确认已有示例问题。
4. 在“实验运行”中选择“历史结果导入”。
5. 上传：

```text
app/examples/demo_historical_results.csv
```

6. 创建实验。
7. 在“评估与失败分析”中点击“开始/重新评估”。
8. 到“导出中心”导出 Excel 和 PDF。

## 上传材料建议

适合上传：

- 数据字典
- 小型样例表
- 业务规则文档
- Markdown 说明
- SQL 建表语句
- 少量脱敏样例数据

暂不适合上传：

- 超大数据库 dump
- 几十 MB 的 Excel
- 大量原始日志
- 未脱敏敏感数据

MVP 只抽样小文件，不做完整文档切分和长期知识库管理。

## LLM 生成测试题注意事项

- API Key 不会保存到数据库。
- 如果公司网络代理较慢，可以把超时调大。
- 生成数量建议先从 20-50 条开始。
- LLM 可能返回格式错误，遇到这种情况可以减少材料长度或换更强的模型。
- 生成后一定要人工审核，不要直接把所有题投入正式评测。

## 使用后的检查清单

- 项目背景是否足够让 LLM 理解业务。
- 上传材料是否过大或信息太杂。
- 测试问题是否可客观判断。
- 参考答案和期望证据是否足够清楚。
- 已通过问题是否才进入实验运行。
- 历史结果是否正确匹配到 `question_id`。
- 外部 API 响应字段是否正确映射。
- 失败标签是否需要人工修正。
- 对比实验的样本范围是否一致。

## 成本提醒

LLM 生成测试集和 LLM 裁判都可能产生费用。评估 1000 条样本可能消耗大量 tokens，因为裁判 prompt 往往包含问题、答案、参考答案和长上下文。建议先小批量验证，MVP 默认单次评估上限为 300 条。
